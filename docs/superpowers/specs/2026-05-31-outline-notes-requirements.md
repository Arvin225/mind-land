# 大纲笔记模块需求文档

**日期：** 2026-05-31  
**状态：** 初稿，已按用户决策更新，待登录态幕布 Web 端补充观察  
**模块：** 大纲笔记（当前路由 `/note`）  

---

## 1. 背景与目标

mind-land 当前已经具备待办、卡片笔记、日记等个人知识管理模块，但“大纲笔记”仍是占位页。用户希望参考幕布 Web 端，建设一个以树状大纲为核心、可一键切换思维导图、并带文件组织、搜索、回收站能力的结构化笔记模块。

本模块目标不是做一个普通富文本编辑器，而是提供“结构优先”的笔记体验：

- 用层级主题承载思考、写作框架、会议记录、学习笔记和计划拆解。
- 用纯键盘完成高频编辑，减少鼠标依赖。
- 同一份树状数据可在“大纲视图”和“思维导图视图”之间切换。
- 文件、文件夹、搜索、回收站形成完整的个人文档管理闭环。

## 2. 信息来源与约束

### 2.1 当前项目基线

- 前端为 React 19 + TypeScript + Vite + Tailwind CSS 4，路由通过 React Router v7 管理。
- 后端为 Go + Gin + GORM + SQLite，已有 `slipbox`、`todo`、`diary` 按 `model/service/handler` 分包。
- `/note` 导航已存在，当前仅为“大纲笔记 Coming soon”占位页。
- `/mindmap` 导航已存在，当前为“脑图 Coming soon”占位页。
- 日记模块已有双栏布局、TipTap 编辑器、自动保存、右键菜单、软删除和回收站实现。
- 卡片笔记已有 TipTap 富文本、图片上传、标签树、卡片菜单和排序入口。
- 待办模块已有拖拽排序和 `Alt + ArrowUp/ArrowDown` 键盘排序模式。
- 全局顶栏已有搜索图标状态 `searchOpen`，但尚未实现实际搜索界面。

### 2.2 幕布公开资料参考

- 幕布官网强调大纲拆解、Markdown 输入、Latex 公式、全场景快捷键、大文档流畅、协作、双向链接和跨平台同步。
- 幕布帮助中心说明 PC/Web 端大纲高频操作包括 `Enter` 新建主题、`Tab` 缩进、`Shift+Tab` 提升层级、拖拽移动、点击三角展开收起。
- 幕布帮助中心说明文档编辑页可通过右上角按钮在大纲和思维导图之间切换，思维导图模式下有缩放、结构、风格等菜单能力。
- 幕布帮助中心说明 PC/Web 端可从左上角 `+` 新建文档，快捷键为 `Alt+N`，文档首页支持文件夹。
- 幕布使用指南说明文档或文件夹可重命名、移动、复制、分享、删除；搜索可查文档或文件夹；回收站可恢复或彻底删除。
- 幕布帮助中心说明大纲模式可导出 Word、PDF、图片、HTML、OPML，思维导图模式可导出图片和 FreeMind，PC/Web 可导入 OPML、FreeMind、XMind；这些能力作为后续参考，不进入 mind-land 第一版范围。
- 幕布帮助中心说明快捷键列表可从文档更多菜单打开，也可用 `Ctrl+/` 或 `Command+/` 打开。
- 幕布帮助中心说明大纲主题菜单存在默认菜单和快速菜单两种触发方式。

本初稿尚未基于用户登录后的实际工作台做细节补充；登录后建议再补一轮：真实文档首页布局、编辑页菜单项、主题悬浮菜单、搜索结果页、回收站页面。

## 3. 产品定位

大纲笔记是 mind-land 的“结构化长笔记”模块，和现有模块的关系如下：

| 模块 | 定位 | 与大纲笔记关系 |
| --- | --- | --- |
| 待办 | 任务执行列表 | 与大纲笔记保持独立；大纲主题第一版不支持待办状态 |
| 卡片笔记 | 碎片化知识卡片 | 大纲适合长结构、框架和线性展开，卡片适合原子知识 |
| 日记 | 时间线记录 | 大纲适合主题性沉淀，不以时间为主轴 |
| 脑图 | 当前占位 | 保留 `/mindmap` 单独导航，作为同一批大纲文档的视觉化入口 |
| 剪藏 | 当前占位 | 后续剪藏内容可保存为大纲节点或卡片 |

建议保留导航标签“大纲笔记”和“脑图”。`/note` 以文件管理和大纲编辑为主，编辑页内仍提供“大纲/思维导图”切换；`/mindmap` 作为单独导航入口，优先展示同一批大纲文档的思维导图视图或最近打开的脑图，不另建一套独立数据模型。

## 4. 用户角色与核心场景

### 4.1 目标用户

- 个人知识管理用户：需要把学习、阅读、项目资料整理成可持续维护的结构。
- 写作者或产品经理：需要先搭框架，再填充细节。
- 学生或备考用户：需要将课程、书籍、知识点梳理成层级大纲与思维导图。
- 日常计划用户：需要把目标拆解成可执行的层级任务。

### 4.2 核心场景

1. 快速创建一篇大纲，连续输入主题，通过 `Enter`、`Tab`、`Shift+Tab` 完成结构组织。
2. 在一个复杂主题中折叠无关分支，只聚焦当前节点及其子节点。
3. 将大纲切换为思维导图，检查结构是否清晰。
4. 在文件夹中管理多篇大纲，按最近编辑、标题、创建时间查找。
5. 搜索某个关键词，跳转到包含关键词的文档和具体主题节点。
6. 删除文档后可在回收站恢复；误删节点可通过撤销恢复。

## 5. 范围与优先级

### P0：第一版必须完成

- 文件夹和文档的基础组织管理。
- 大纲文档创建、重命名、移动、复制、删除。
- 大纲主题树编辑：新增、缩进、提升、删除、拖拽、展开收起、聚焦进入主题。
- 快捷键：`Enter`、`Tab`、`Shift+Tab`、撤销/重做、加粗/斜体/下划线、快捷键列表。
- 主题菜单：新增子主题、复制、移动、聚焦、删除、格式化。
- 基础 Markdown 输入转换。
- 大纲和思维导图视图切换。
- 全局文档搜索和文档内搜索。
- 文档/文件夹回收站：恢复、彻底删除、清空回收站。
- 自动保存和保存状态提示。
- 文档历史版本：查看历史快照，并从历史版本恢复。

### P1：增强体验

- OPML 导入/导出，Markdown 导入/导出。
- 思维导图图片导出。
- 主题颜色、图标或 Emoji。
- `@` 人名或标签检索，沿用卡片笔记已有 `#tag` 习惯。
- 批量选择主题、批量移动和批量删除。
- 高级排序、更多筛选条件和自定义视图偏好。

### P2：高级能力

- XMind/FreeMind 导入导出。
- 双向链接、反向链接和引用面板。
- 演示模式。
- 多端同步或协作编辑。
- AI 辅助生成大纲、总结、改写和一键转脑图。

## 6. 信息架构与页面结构

### 6.1 模块入口

当前 `/note` 路由作为大纲笔记入口。建议页面内部采用“三层导航”：

```text

    └── 主工作区
        ├── 未选中文档：文档列表或网格
        └── 已选中文档：大纲编辑器或思维导图
```

### 6.2 文档管理视图

文档管理视图参考幕布“文档首页”，但适配 mind-land 现有窄侧栏风格：

- 左侧二级栏：
  - 全部文档
  - 最近编辑
  - 收藏
  - 回收站
  - 文件夹树
- 顶部操作区：
  - 新建文档
  - 新建文件夹
  - 搜索
  - 排序
  - 列表/网格切换
- 主区域：
  - 文件夹卡片或列表项
  - 文档卡片或列表项
  - 空状态和加载状态

```text

├─────────────────────────────────────────────────────────────┤
```

编辑器顶部保留：

## 7. 功能需求

### 7.1 文件组织管理

#### 7.1.1 新建

- 用户可在大纲笔记模块中新建文档和文件夹。
- 新建文档默认位于当前文件夹。
- 新建文档后自动打开编辑视图，文档标题默认为“未命名大纲”。
- 支持快捷键 `Alt+N` 新建文档；Mac 对应 `Option+N`。

#### 7.1.2 文档和文件夹操作

每个文档和文件夹应提供菜单：

- 打开
- 重命名
- 移动到
- 复制
- 收藏/取消收藏
- 删除到回收站

文件夹额外支持：

- 新建子文档
- 新建子文件夹
- 展开/收起

第一版不做分享，因为 mind-land 当前是本地个人工具，没有账号、权限和公开链接系统。

#### 7.1.3 排序与视图

列表支持排序：

- 最近编辑优先
- 创建时间
- 标题 A-Z / Z-A
- 自定义排序

视图支持：

- 列表视图：适合大量文档扫描。
- 网格视图：适合少量文件夹和视觉浏览。

#### 7.1.4 最近编辑和收藏

- 最近编辑为虚拟视图，不改变文件夹结构。
- 收藏为文档/文件夹上的布尔属性。
- 最近编辑列表不显示文件夹层级，但每条结果应显示所在路径。

### 7.2 大纲主题编辑

#### 7.2.1 主题数据结构

每篇文档由一个根文档和多条主题节点组成。主题节点至少包含：

- `id`
- `docId`
- `parentId`
- `sortOrder`
- `content`
- `plainText`
- `collapsed`
- `note`
- `createdAt`
- `updatedAt`
- `metadata`

节点结构必须天然支持树形渲染和思维导图渲染，不能只保存 HTML。

#### 7.2.2 基础编辑

用户可完成：

- 在当前主题下方新建同级主题。
- 在当前主题下新建子主题。
- 缩进主题，使其成为上一同级主题的子主题。
- 提升主题，使其回到父级的同级。
- 删除主题及其子树。
- 拖拽主题调整位置和层级。
- 折叠或展开有子主题的节点。
- 聚焦进入某个主题，仅显示该主题及其子树。
- 通过面包屑返回上一级或文档根部。

#### 7.2.3 主题菜单

主题圆点或菜单按钮应支持：

- 新增同级主题
- 新增子主题
- 进入主题
- 展开/收起子主题
- 复制主题文本
- 复制主题及子主题
- 移动到
- 上移/下移
- 缩进/提升
- 添加备注
- 设置颜色或图标
- 删除

菜单触发方式：

- 默认：悬浮到主题圆点附近显示快捷入口。
- 快速菜单：可在编辑设置里启用，点击主题左侧菜单按钮直接打开。

#### 7.2.4 多选与批量操作

P0 可先支持单节点操作。P1 增加：

- Shift 连续选择。
- Ctrl/Command 多选。
- 批量拖拽。
- 批量缩进、提升、删除、复制。

### 7.3 快捷键

第一版快捷键应以大纲编辑效率为中心：

| 快捷键 | 功能 |
| --- | --- |
| `Enter` | 新建同级主题 |
| `Shift+Enter` | 在当前主题内换行或添加备注，具体行为在设置中可切换 |
| `Tab` | 缩进一级 |
| `Shift+Tab` | 提升一级 |
| `Backspace` | 空主题时删除并聚焦上一主题；非空时正常删除字符 |
| `Delete` | 删除选中主题 |
| `Ctrl/Command+Z` | 撤销 |
| `Ctrl/Command+Shift+Z` | 重做 |
| `Ctrl/Command+B` | 加粗 |
| `Ctrl/Command+I` | 斜体 |
| `Ctrl/Command+U` | 下划线 |
| `Alt+ArrowUp` | 当前主题上移 |
| `Alt+ArrowDown` | 当前主题下移 |
| `Ctrl/Command+[` | 从聚焦主题返回上一级 |
| `Ctrl/Command+F` | 文档内搜索 |
| `Ctrl/Command+/` | 打开快捷键列表 |
| `Alt+N` / `Option+N` | 新建文档 |

快捷键列表应通过更多菜单和 `Ctrl/Command+/` 打开。

### 7.4 Markdown 与富文本

#### 7.4.1 P0 Markdown 输入

支持在主题内输入 Markdown 触发格式转换：

- `# `、`## `、`### ` 转标题层级样式。
- `- `、`* ` 转普通主题或列表样式。
- `1. ` 转有序列表样式。
- `> ` 转引用样式。
- `**text**` 转加粗。
- `*text*` 或 `_text_` 转斜体。
- `~~text~~` 转删除线。
- `` `code` `` 转行内代码。
- `[text](url)` 转链接。
- `![alt](url)` 转图片。
- `$formula$` 转 Latex 行内公式。

第一版不提供 Markdown 源码模式，也不把 `- [ ]` / `- [x]` 转换为待办主题。

#### 7.4.2 粘贴处理

- 粘贴多行纯文本时，每行生成一个同级主题。
- 粘贴带缩进的 Markdown 时，缩进应转换为层级。
- 粘贴 HTML 时清洗不安全标签，保留可支持的基础格式。
- 第一版不处理 OPML、XMind、FreeMind 等导入格式。

#### 7.4.3 存储原则

大纲主题不应只保存 TipTap HTML。推荐保存：

- `contentJson`：编辑器富文本结构，便于恢复样式。
- `plainText`：搜索和摘要。
- `markdown`：第一版不作为源码编辑内容，仅可作为后续导出和互操作的预留字段。

### 7.5 大纲与思维导图切换

#### 7.5.1 切换规则

- 大纲视图和思维导图视图共享同一份主题树数据。
- 切换视图不应触发数据转换丢失。
- 当前聚焦主题、展开状态、选中节点应尽量在两个视图之间保持。

#### 7.5.2 思维导图 P0 能力

- 根据主题树自动布局。
- 支持缩放、拖动画布、适应屏幕、回到中心。
- 支持展开/收起子树。
- 点击节点可选中，双击或回车回到大纲定位该主题。
- 顶部可切回“大纲笔记”。

#### 7.5.3 思维导图 P1 能力

- 直接在图上编辑节点文本。
- 拖动节点调整结构。
- 切换结构：右侧树、左右发散、组织结构图。
- 切换主题风格：简洁、彩色、紧凑。
- 导出图片。

### 7.6 搜索

#### 7.6.1 全局搜索

全局顶栏的搜索图标应打开搜索面板，覆盖所有大纲文档：

- 搜索文档标题。
- 搜索文件夹名称。
- 搜索主题内容。
- 搜索 `#tag` 和 `@person`。
- 展示命中文档、所在路径、命中主题摘要。
- 点击结果打开文档并定位到主题。

P0 可使用 SQLite `LIKE` 和前端高亮；P1 再考虑 FTS 或独立索引表。

#### 7.6.2 文档内搜索

文档内 `Ctrl/Command+F` 打开小型搜索条：

- 高亮所有命中主题。
- 上一个/下一个结果跳转。
- 展开包含命中的折叠父节点。
- 在思维导图视图中搜索时，定位并居中节点。

### 7.7 回收站

#### 7.7.1 范围

P0 回收站覆盖文档和文件夹。大纲主题节点删除通过撤销恢复，不进入回收站。P2 可考虑节点级回收站。

#### 7.7.2 删除规则

- 删除文档：设置 `del=true`，记录删除时间和原父文件夹。
- 删除文件夹：文件夹及其子文档/子文件夹整体进入回收站。
- 删除前需要确认，文案明确“可在回收站恢复”。

#### 7.7.3 回收站视图

回收站展示：

- 名称
- 类型：文档或文件夹
- 原路径
- 删除时间
- 操作：恢复、彻底删除

支持：

- 单项恢复
- 单项彻底删除
- 清空回收站

恢复规则：

- 原父文件夹存在时恢复到原位置。
- 原父文件夹已被彻底删除时恢复到根目录，并提示用户。

### 7.8 文档历史版本

第一版需要支持文档级历史版本，用来降低自动保存和复杂结构编辑带来的误操作风险。

#### 7.8.1 版本生成

- 文档创建时生成初始版本。
- 自动保存时不需要每次都生成新版本，避免版本爆炸。
- 建议按策略生成历史快照：首次保存、标题变更、距离上次版本超过 5 分钟且内容有变化、用户手动创建版本。
- 版本快照保存整篇文档的元信息和节点树结构。

#### 7.8.2 版本查看

更多菜单中提供“历史版本”入口。历史版本面板展示：

- 版本时间
- 版本来源：自动保存 / 手动保存 / 标题变更 / 恢复前备份
- 主题数量
- 简短摘要

用户可打开某个历史版本进行只读预览。

#### 7.8.3 版本恢复

- 用户可将当前文档恢复到某个历史版本。
- 恢复前需要二次确认。
- 执行恢复前应自动创建一份“恢复前备份”，避免覆盖当前内容后无法回退。
- 恢复只影响该文档标题和节点树，不影响文件夹位置、收藏状态、回收站状态。

## 8. 技术方案建议

### 8.1 三种方案

#### 方案 A：自研树状大纲编辑器，复用局部 TipTap 能力（推荐）

为大纲主题树实现专用 React 组件和快捷键系统；每个主题的富文本内容可使用轻量 contenteditable 或 TipTap 单节点编辑。

优点：

- 最贴合大纲语义，缩进、提升、聚焦、拖拽和思维导图转换更可控。
- 数据结构清晰，方便搜索、导入导出、回收站和未来双向链接。
- 历史版本可以直接保存节点树快照，恢复逻辑清晰。
- 与现有 Redux + Go API 模式容易融合。

缺点：

- 编辑器行为需要更多自研测试，尤其是输入法、选区、撤销栈。

#### 方案 B：基于 TipTap/ProseMirror 定制大纲 Node

将整篇大纲作为一个 ProseMirror 文档，通过自定义节点表达主题树。

优点：

- 富文本、Markdown、撤销重做、选区能力更成熟。
- 与现有 TipTap 经验一致。

缺点：

- 树结构操作、节点拖拽、跨视图同步和持久化复杂度较高。
- 后续思维导图和节点级搜索需要从编辑器文档中解析结构。

#### 方案 C：引入现成大纲/脑图库

用第三方库快速实现大纲或脑图，再适配 mind-land 数据模型。

优点：

- 初期速度快。
- 思维导图布局可能更快成型。

缺点：

- UI 风格、快捷键、数据结构和导入导出很容易受库限制。
- 长期维护风险较大。

### 8.2 推荐结论

推荐方案 A。大纲笔记的核心不是富文本，而是稳定的树操作和键盘操作。mind-land 当前已有 TipTap 使用经验，但大纲模块应以“树节点模型”为核心，再将富文本作为节点内容的一部分。

## 11. 非功能需求

### 11.1 性能

- 1000 个主题节点以内编辑、折叠、搜索应无明显卡顿。
- 5000 个主题节点以内可打开和浏览，允许局部虚拟化。
- 自动保存应 debounce，建议 800-1500ms；离开页面前确保最后一次保存完成。
- 思维导图视图应按可见节点渲染，折叠子树不参与布局。

### 11.2 可访问性与键盘体验

- 大纲树应支持键盘导航。
- 当前焦点主题必须有清晰可见的焦点态。
- 菜单、弹窗、快捷键面板支持 `Esc` 关闭。
- 按钮有 `aria-label` 和 tooltip。

### 11.3 数据安全

- 自动保存失败时显示错误状态，并允许重试。
- 删除和清空回收站必须二次确认。
- 粘贴和渲染 HTML 时使用 DOMPurify 或等价安全清洗。
- 前端对节点树变更做本地撤销栈，减少误操作损失。
- 历史版本保存恢复前备份，避免恢复操作本身造成不可逆覆盖。

### 11.4 平台范围

- 第一版优先 Web 端桌面浏览器体验。
- 移动端仅保证基础可打开，不作为第一版验收重点。

## 12. 验收标准

### 12.1 文件组织

- 可以新建文件夹和大纲文档。
- 可以将文档移动到指定文件夹。
- 可以复制、重命名、删除文档。
- 最近编辑和收藏视图能正确显示文档。

### 12.2 大纲编辑

- `Enter` 连续创建主题。
- `Tab` 和 `Shift+Tab` 能调整层级，且非法缩进被阻止。
- 主题拖拽后顺序和层级持久化。
- 折叠父主题后隐藏子树，重新打开文档仍保持折叠状态。
- 进入主题后只显示该主题子树，可通过面包屑返回。
- 主题菜单各项操作生效。

### 12.3 Markdown 和快捷键

- 基础 Markdown 输入能转换为对应格式。
- `Ctrl/Command+B/I/U` 能修改选中文本或当前输入格式。
- `Ctrl/Command+/` 能打开快捷键列表。
- 撤销和重做覆盖节点新增、删除、移动、文本编辑。

### 12.4 思维导图

- 同一文档可从大纲切换到思维导图。
- 思维导图能展示与大纲一致的层级。
- 折叠、选中、聚焦的状态能合理映射。
- 可缩放、拖动画布、回到中心。
- 可返回大纲并定位到当前节点。

### 12.5 搜索

- 全局搜索能命中文档标题、文件夹名称和主题文本。
- 搜索结果能打开文档并定位主题。
- 文档内搜索能展开父节点并高亮命中主题。

### 12.6 历史版本

- 文档编辑一段时间后能生成历史版本。
- 历史版本列表能展示版本时间、来源、摘要和主题数量。
- 可以只读预览某个历史版本。
- 恢复历史版本前有确认提示。
- 恢复前自动生成“恢复前备份”。

### 12.7 回收站

- 删除文档或文件夹后不在正常列表出现。
- 回收站可看到删除项和原路径。
- 恢复后回到原位置；原位置不存在时回到根目录。
- 彻底删除和清空回收站后数据不可恢复。

## 13. 已确认决策

1. `/mindmap` 保留为单独导航，但与大纲笔记共享文档和节点数据。
2. 第一版不支持导入/导出；OPML、Markdown、图片等导入导出能力放入 P1 或后续。
3. 大纲主题第一版不支持待办状态，也不同步到待办模块。
4. 第一版需要文档历史版本。
5. 第一版优先 Web 端桌面浏览器体验。
6. Markdown 仅支持输入转换，不提供 Markdown 源码模式。
