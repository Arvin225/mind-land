# Outline 文件管理系统重设计

参考幕布，实现传统文件浏览器布局。

## 当前问题

1. 左侧导航「全部文档/收藏/最近/回收站」和文件夹树是分离的
2. 没有统一的根目录概念
3. 进入编辑器后左侧 sidebar 消失
4. 面包屑只在 focus mode 出现，编辑时顶部没有路径导航
5. 全局搜索按钮多余

## 目标布局

```
┌──────────────────────────────────────────────────────┐
│ 面包屑：我的文档 > 文件夹A > 文件夹B          [操作] │  ← 始终可见
├──────────────┬───────────────────────────────────────┤
│ 左侧 280px   │  右侧内容区                           │
│              │                                       │
│ ▼ 我的文档   │  [新建文档] [新建文件夹]              │
│   ▼ 文件夹A  │                                       │
│     📄 文档1 │  ┌────┐ ┌────┐ ┌────┐               │
│     📄 文档2 │  │文档│ │文档│ │文档│               │
│   ▶ 文件夹B  │  └────┘ └────┘ └────┘               │
│   ▶ 文件夹C  │                                       │
│              │                                       │
│ 收藏         │                                       │
│ 最近         │                                       │
│ 回收站       │                                       │
│              │                                       │
└──────────────┴───────────────────────────────────────┘
```

## 详细规格

### 1. 面包屑 (Breadcrumb)

- **位置**：页面顶部，左侧 sidebar 和右侧内容区之上，横跨全宽
- **显示**：`我的文档 > 文件夹A > 文件夹B`，每段可点击跳转
- **状态**：
  - 在「我的文档」根目录：只显示「我的文档」（不可点击，或点击无效果）
  - 在子文件夹中：显示完整路径，每段可点击回到对应层级
  - 在文档编辑器中：面包屑常驻，格式 `我的文档 > 文件夹A > 文档标题`，点击「我的文档」或文件夹返回对应视图
  - 「最近」「收藏」「回收站」视图：显示对应标签而非路径
- **实现**：独立组件 `BreadcrumbBar`，放在 `Note/index.tsx` 最顶部

### 2. 左侧目录树 (FolderTreePanel)

**默认根节点：「我的文档」**
- 始终存在，不可删除/重命名
- 选中「我的文档」时右侧显示所有根级文件夹 + 根级文档（folderId=0）

**子节点**：展开的文件夹 + 其下文档
- 文件夹可展开/折叠
- 文件夹内的文档直接显示在文件夹下方（缩进一级）
- 点击文件夹 → 右侧显示该文件夹内容 + 面包屑更新
- 点击文档 → 进入编辑器

**选中态**：
- 当前在哪个文件夹就高亮哪个
- 在编辑器时，文档所属文件夹高亮（文档名也高亮？按传统文件管理器，编辑时左侧选中文档本身或所属文件夹均可，这里选中文档本身更直观）

**底部固定导航**（不变）：
- 收藏 (Star)
- 最近 (Clock)  
- 回收站 (Trash2)

### 3. 右侧内容区

**当选中文件夹时**：
- 顶部工具栏：新建文档、新建文件夹按钮
- 文件列表（grid 或 list 视图）：
  - 文件夹在上，文档在下
  - 每个项有图标 + 名称
  - 点击文件夹 → 进入该文件夹
  - 点击文档 → 打开编辑器
  - 右键菜单

**当选中「收藏」「最近」「回收站」时**：
- 对应的文档列表（保持现有行为）

### 4. 编辑器视图的变化

- 面包屑常驻（顶部）
- 左侧目录树保持可见（不收起）
- 当前打开的文档在左侧树中高亮
- 编辑器占据右侧区域（左侧树之外的全部空间）

### 5. 移除的功能

- 顶部全局搜索按钮
- DocumentList 中的局部搜索（可选保留）

## 状态管理变更

### Redux State 新增字段

```typescript
currentFolderId: number | null;  // 当前选中的文件夹（null=我的文档根）
breadcrumbPath: { id: number; name: string }[];  // 面包屑路径
```

### 视图模式

- `home` + `currentFolderId=null` → 我的文档根
- `home` + `currentFolderId=N` → 文件夹 N 的内容
- `editor` → 文档编辑器（左侧树+面包屑仍可见）

## 组件树变更

```
Note/index.tsx

├── (双栏布局)
│   ├── FolderTreePanel.tsx    ← 重构 FolderSidebar

│   └── (右侧)
│       ├── DocumentHome.tsx   ← 重构为 FolderContent
│       │   ├── ContentToolbar.tsx      ← 新建按钮行
│       │   └── ContentList.tsx          ← 文件夹+文档列表
│       └── OutlineEditor.tsx  ← 嵌入右侧（不再全屏）
```

## 实现步骤

1. **Redux 状态变更**：添加 currentFolderId、breadcrumbPath、selectedItem
2. **BreadcrumbBar**：新建组件
3. **FolderTreePanel**：重构，加入「我的文档」根节点、文档子节点、选中态同步
4. **右侧内容区重构**：FolderContent 替换 DocumentHome
5. **编辑器嵌入**：编辑器不再全屏，左侧树保持可见
6. **移除全局搜索**：删掉 GlobalSearchPanel 入口
7. **API 调整**（如需要）：确保 folderId=0 的文档可查询
